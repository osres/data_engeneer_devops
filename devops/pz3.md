# Практическое задание

## Тема

**Docker: работа с образами и контейнерами**

## Цель

Закрепить базовые навыки работы с Docker:

- находить и загружать образы;
- просматривать список образов;
- создавать и запускать контейнеры;
- останавливать, запускать повторно и удалять контейнеры;
- анализировать состояние контейнеров;
- работать с логами и выполнять команды внутри контейнера.

## Задание (шаги)

### Этап 1. Проверка установки Docker

Проверьте, что Docker установлен и работает корректно.

**Что нужно сделать:**

1. Проверить версию Docker.
2. Убедиться, что Docker daemon запущен.
3. Посмотреть общую информацию о системе Docker.

### Подсказки по ключевым частям

Возможные команды:

- `docker --version`
  Показывает установленную версию Docker CLI.
- `docker version`
  Показывает подробную информацию о клиенте и сервере Docker.
- `docker info`
  Выводит информацию о Docker Engine: количество контейнеров, образов, драйвер хранения и другие параметры.

### Этап 2. Поиск и загрузка образа

Найдите в Docker Hub официальный образ и загрузите его локально.

**Что нужно сделать:**

1. Найти образ `nginx`.
2. Скачать его на локальную машину.
3. Проверить, что образ появился в списке локальных образов.

### Подсказки по ключевым частям

Возможные команды:

- `docker search nginx`
  Ищет образы по ключевому слову в реестре.
- `docker pull nginx`
  Загружает образ `nginx` из Docker Hub.
- `docker images`
  Показывает список локально доступных образов.
- `docker image ls`
  Аналог `docker images`, показывает локальные образы.

### Этап 3. Запуск контейнера из образа

Создайте и запустите контейнер на основе скачанного образа.

**Что нужно сделать:**

1. Запустить контейнер из образа `nginx`.
2. Назначить контейнеру понятное имя.
3. Пробросить порт контейнера на порт хост-машины.
4. Проверить, что контейнер работает.

### Подсказки по ключевым частям

Возможные команды:

- `docker run nginx`
  Создаёт и запускает контейнер из образа `nginx`.
- `docker run --name mynginx -p 8080:80 -d nginx`
  Создаёт контейнер с именем `mynginx`, публикует порт `80` контейнера на `8080` хоста и запускает его в фоновом режиме.
- `docker ps`
  Показывает только запущенные контейнеры.
- `docker ps -a`
  Показывает все контейнеры, включая остановленные.

### Этап 4. Проверка работы контейнера

Убедитесь, что приложение внутри контейнера действительно доступно.

**Что нужно сделать:**

1. Проверить контейнер через браузер или командную строку.
2. Убедиться, что страница `nginx` открывается.
3. Посмотреть логи контейнера.

### Подсказки по ключевым частям

Возможные команды:

- `docker logs mynginx`
  Показывает логи контейнера `mynginx`.
- `docker logs -f mynginx`
  Показывает логи в режиме реального времени.
- `curl http://localhost:8080`
  Отправляет HTTP-запрос к локальному порту 8080 для проверки ответа сервиса.
- `docker port mynginx`
  Показывает, какие порты контейнера опубликованы на хосте.

### Этап 5. Работа с состоянием контейнера

Научитесь управлять жизненным циклом контейнера.

**Что нужно сделать:**

1. Остановить контейнер.
2. Проверить, что он остановился.
3. Снова запустить контейнер.
4. Перезапустить контейнер одной командой.

### Подсказки по ключевым частям

Возможные команды:

- `docker stop mynginx`
  Останавливает контейнер.
- `docker start mynginx`
  Запускает ранее остановленный контейнер.
- `docker restart mynginx`
  Перезапускает контейнер.
- `docker ps`
  Показывает запущенные контейнеры.
- `docker ps -a`
  Показывает все контейнеры и их статусы.

### Этап 6. Получение информации о контейнере

Изучите свойства созданного контейнера.

**Что нужно сделать:**

1. Посмотреть подробную информацию о контейнере.
2. Определить его статус, IP-адрес, имя, используемый образ.
3. Проверить настройки портов.

### Подсказки по ключевым частям

Возможные команды:

- `docker inspect mynginx`
  Показывает подробную техническую информацию о контейнере в формате JSON.
- `docker inspect --format='{{.State.Status}}' mynginx`
  Показывает только статус контейнера.
- `docker inspect --format='{{.NetworkSettings.IPAddress}}' mynginx`
  Показывает IP-адрес контейнера.
- `docker container ls`
  Показывает список активных контейнеров.

### Этап 7. Выполнение команд внутри контейнера

Подключитесь к контейнеру и выполните команды внутри него.

**Что нужно сделать:**

1. Запустить команду внутри работающего контейнера.
2. Проверить содержимое файлов веб-сервера.
3. При наличии shell — открыть интерактивную сессию.

### Подсказки по ключевым частям

Возможные команды:

- `docker exec mynginx ls /usr/share/nginx/html`
  Выполняет команду `ls` внутри контейнера.
- `docker exec -it mynginx sh`
  Запускает интерактивную shell-сессию внутри контейнера.
- `docker exec mynginx cat /usr/share/nginx/html/index.html`
  Показывает содержимое главной страницы внутри контейнера.

### Этап 8. Создание собственного образа

Создайте простой кастомный образ на основе `nginx`.

**Что нужно сделать:**

1. Создать папку проекта.
2. Подготовить собственный файл `index.html`.
3. Создать `Dockerfile`, который использует образ `nginx` и копирует ваш файл в контейнер.
4. Собрать собственный образ.
5. Запустить контейнер на основе этого образа.
6. Проверить, что в браузере отображается ваша страница.

### Подсказки по ключевым частям

Возможные команды:

- `docker build -t mynginx:v1 .`
  Собирает образ из `Dockerfile` в текущей директории и присваивает тег `mynginx:v1`.
- `docker run --name mysite -p 8081:80 -d mynginx:v1`
  Запускает контейнер из собственного образа.
- `docker images`
  Проверяет, что новый образ создан.
- `docker ps`
  Проверяет, что контейнер запущен.
- `curl http://localhost:8081`
  Проверяет содержимое страницы.

Пример минимального `Dockerfile`:

```dockerfile
FROM nginx
COPY index.html /usr/share/nginx/html/index.html
```

Пример простого `index.html`:

```html
<h1>Мой Docker-сайт работает</h1>
```

### Этап 9. Удаление контейнеров и образов

Очистите окружение после выполнения работы.

**Что нужно сделать:**

1. Остановить созданные контейнеры.
2. Удалить контейнеры.
3. Удалить собственный образ.
4. При желании удалить скачанный образ `nginx`.

### Подсказки по ключевым частям

Возможные команды:

- `docker stop mynginx mysite`
  Останавливает один или несколько контейнеров.
- `docker rm mynginx mysite`
  Удаляет контейнеры.
- `docker rmi mynginx:v1`
  Удаляет образ.
- `docker rmi nginx`
  Удаляет образ `nginx`, если он не используется контейнерами.
- `docker container prune`
  Удаляет все остановленные контейнеры.
- `docker image prune`
  Удаляет неиспользуемые образы.

## Что проверить перед отправкой (чек-лист)

Проверьте, что у вас выполнено следующее:

- Docker установлен и команды выполняются без ошибок.
- Образ `nginx` был найден и загружен.
- Контейнер из образа `nginx` был успешно запущен.
- Порт контейнера был опубликован на хост.
- Страница `nginx` открывалась в браузере или через `curl`.
- Логи контейнера были просмотрены.
- Контейнер был остановлен и снова запущен.
- Была получена информация о контейнере через `docker inspect`.
- Была выполнена хотя бы одна команда внутри контейнера через `docker exec`.
- Создан собственный `Dockerfile`.
- Собран собственный образ.
- Контейнер из собственного образа успешно запущен.
- В браузере или через `curl` отображалась ваша собственная HTML-страница.
- После завершения работы ненужные контейнеры и образы удалены.

## Советы по улучшению работы

1. **Используйте осмысленные имена контейнеров**
   Вместо случайных идентификаторов задавайте имена через `--name`, например `web1`, `test-nginx`, `site-demo`.

2. **Добавляйте теги к образам**
   Например, `mynginx:v1`, `mynginx:v2`. Это помогает различать версии.

3. **Фиксируйте результаты команд**
   Сделайте скриншоты или сохраните вывод команд:
   - `docker images`
   - `docker ps`
   - `docker logs <container_name>`
   - `curl http://localhost:<port>`

4. **Сравнивайте образы и контейнеры**
   В отчёте кратко укажите:
   - образ — это шаблон;
   - контейнер — это запущенный экземпляр образа.

5. **Попробуйте дополнительные эксперименты**
   - запустить несколько контейнеров из одного образа;
   - использовать другой внешний порт;
   - изменить `index.html` и пересобрать образ;
   - посмотреть разницу между `run`, `start`, `exec`, `logs`, `inspect`.
